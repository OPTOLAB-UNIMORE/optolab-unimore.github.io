# OptoLAB content verification checklist

Complete this checklist before the website is described as final or used for institutional communication.

## Identity and governance

- [ ] Confirm the official laboratory name: **OptoLAB**
- [ ] Confirm or replace the provisional extended title: **Optical and Optoelectronic Biomedical Instrumentation Laboratory**
- [ ] Approve the institutional laboratory description and tagline
- [ ] Confirm the responsible UNIMORE department
- [ ] Obtain an authorized institutional or laboratory logo and usage guidance
- [ ] Confirm who approves and maintains website content
- [ ] Review privacy requirements and add an appropriate notice if required

## Contact information

- [ ] Verify the full institutional address
- [ ] Verify the public institutional email address
- [ ] Verify any public telephone number
- [ ] Confirm map provider/link and privacy implications
- [ ] Confirm the correct institutional web links

## Team

For every member, collect and approve:

- [ ] Name
- [ ] Academic role
- [ ] Short biography
- [ ] Research interests
- [ ] Institutional profile URL
- [ ] ORCID
- [ ] Google Scholar URL
- [ ] GitHub URL
- [ ] Authorized, accessible profile photo

## Research and outputs

- [ ] Replace demonstrative project cards with verified publishable projects
- [ ] Confirm which research areas are current and how they should be described
- [ ] Add verified publications and check all BibTeX metadata
- [ ] Verify funding acknowledgements, grant identifiers and required wording
- [ ] Verify collaborations and obtain permission before naming partners
- [ ] Verify patents or intellectual-property statements before publication
- [ ] Review every quantitative result, performance claim and clinical statement
- [ ] Review public software descriptions against current repository READMEs

## Facilities and media

- [ ] Create a verified equipment inventory
- [ ] Approve equipment names, models, capabilities and access conditions
- [ ] Select laboratory, team, facility and prototype photographs
- [ ] Record copyright, licence or written authorization for every image
- [ ] Obtain consent or releases for identifiable people where required
- [ ] Write meaningful alternative text for every informative image
- [ ] Confirm that no confidential prototypes, screens or data appear in photos

## Release review

- [ ] Check English copy with the laboratory
- [ ] Confirm future Italian localization approach
- [ ] Test navigation, search and keyboard use
- [ ] Review color contrast and alternative text
- [ ] Check the mobile layout on representative devices
- [ ] Verify all internal and external links
- [ ] Confirm canonical URL, social preview and favicon
- [ ] Confirm GitHub Pages and custom-domain ownership
